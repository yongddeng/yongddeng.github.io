## I
---
The [analytical engine]() (1837), designed by English mathematician Charles Babbage, is regarded as the concept of a general-purpose digital computer. The machine featured an arithmetic logic unit, control flow with conditional branching and loops, and memory modules. A [Turing machine]() (1936), proposed by Alan Turing, established a theoretical framework to analyse computational capabilities and limitations, laying the foundation for modern Computer Science. Before long, the [electronic numerical integrator and computer](), referred to as ENIAC (1945), designed by John Mauchly and J. Presper Eckert, became the first programmable, electronic, general-purpose digital computer.

While computers vary in size, processing power, and intended purposes, they generally fall into one of the following categories: i) [personal computers]() (PCs): extremely versatile and typically used by individuals for a wide range of office works, web browsing, gaming, multimedia, emulation, programming, and etc; ii) [mainframes]():  designed for high-volume transaction processing, bulk data handling, and mission-critical system that demand exceptional reliability and security; iii) [supercomputers](): deliver extraordinary processing speed and computational power, ideal for intensive tasks such as climate modeling, simulations, and intricate computations in fields like physics and cryptography.

The [motherboard]() inside a computer serves the physical circuitries and connections for three key components of [Von Neumann architecture]() proposed in 1940s. This is the central hub where the CPU socket, RAM slots, storage interfaces, I/O ports, and other peripherals seamlessly communicate via the [buses]() and [chipsets](). The module in modern computers have expanded slots for GPUs, sound cards, or extra storages, intended to offer capabilities or flexibilities for future upgrades. The BIOS/UEFI [firmware]() embedded into the module first performs [self-tests]() (POST) as the power supplies, then the bootloader (e.g. Linux's GRUB, Windows's Boot Manager) loads the kernel from the SSD into RAM.

## II
---
**1. Central Processing Unit**

The [central processing unit]() (CPU or processor) executes a set of instructions, called a [program](), by reading and writing values in registers or memory, potentially altering their states. [Registers]() are the fastest memory, storing only the immediate instructions and data, and are located closest to the CPU's processing units. Given that each [core]() processes with [arithmetic logic unit]() (ALU) and [control unit]() (CU), and thereby the clock speed (GHz) and core count matter, modern CPUs hold between 4 to 64 cores for concurrent and parallel processing. Besides, a CPU also equips a multi-level [cache]()—L1 and L2 within each core, and shared L3—located closer to the cores, aiming to reduce memory latency.

**2. Random Access Memory**

A computer memory is basically an array of bytes each with a unique address. That is, an [random access memory]() (RAM), register, and cache can be viewed as unique abstractions. [Dynamic RAM]() (DRAM) latency depends on data access patterns and physical design of memory cells. Access is fastest when data is within an active row (i.e. contiguous), but latency increases by $T_{\text{precharge}} + T_{\text{row activation}} + T_{\text{column access}}$ as a new row must be accessed. [Static RAM]() (SRAM), known for its high speed and stability despite its higher cost, is therefore appropriate for cache. Notice that a portion of persistent storage could be allocated as [virtual memory]() when a process exceeds the memory limits of RAM.


**3. Persistent Storage**

Persistent storage includes [hard disk drives]() (HDDs) and [solid-state drives]() (SSDs), both non-volatile, retaining data without a power supply. HDDs are mechanical devices that read data through a process requiring physical movement to access an address, leading to slower access speeds. In contrast, SSDs utilise electronic signals, resulting in significantly faster access. In particular, many products released from 2016 onwards come equipped with [non-volatile memory express]() (NVMe) SSDs, in which NVMe operates over the [PCI Express]() (PCIe) interface to allow direct communication between the SSDs and the CPU, or, use flash memory that operates similarly to SSDs for storing data and Apps.

**<a name="gpu"></a>4. Graphics Processing Unit**

The [graphics processing unit]() (GPU), an integral of a graphics card, is designed for [single instruction, multiple threads]() (SIMT) execution of relatively simple and repetitive tasks. Unlike CPUs, GPUs have a way more cores while each operates at a lower clock speed, but still ensure efficient computation and high-throughput, when paired with high-bandwidth memory. Unfortunately, the similar terms [floating point operations per second]() (FLOPS) and [floating ooint operations]() (FLOPs) are used respectively to quantify a GPU’s performance and a deep learning model's computational demands, with the latter often varying based on hardware and model design (e.g. #. of params).

- NVIDIA GPUs are fundamentally built to handle multithreading, particularly through their parallel architecture. NVIDIA GPUs achieve this primarily with their SIMT (Single Instruction, Multiple Threads) model, where many threads can execute the same instruction in parallel on different data. 
- This is essential for handling large-scale parallel workloads, which is why GPUs are so effective for tasks like machine learning, graphics rendering, and scientific computing. Unlike traditional CPU threads, these GPU threads are lightweight and managed by hardware. The SM’s scheduler automatically handles warp-level context switching.

The [streaming multiprocessor]() (SM) within a GPU is a core processing unit responsible for executing thread blocks. Given that a [thread]() is the smallest unit of execution which might handle part of a tensor operation (e.g. an element-wise multiplication), threads are grouped into [wraps]() of equal size to execute instructions simultaneously under the SIMT model, and a [thread block]() is a collection of warps assigned to an SM. To prevent underutilisation of SMs (e.g. *"a single thread block on a single SM"* scenario), we should dispatch more thread blocks than the number of existing SMs, and the [warp scheduler]() leads SMs to switch between, the less the impact of latency, maximising its throughput.

- An [NVIDIA's architect](https://www.youtube.com/watch?v=n6M8R8-PlnE&t=75s) explained that the memory bandwidth is what actually matters, not a peak TFLOPS.
- Each SMs (in A100) can load 64 bytes per clock cycle, and 108 SMs running at 1410 MHz, that is 64 * 108 * 1410 = 9750 GB/s. However, HBM2 memory bandwidth is only 1555 GB/s, thus we cores are requesting 6.3x more than what the memory can deliver.
  
A [cuda core]() within an SM is a a general-purpose processing units, essentially an ALU, performing basic arithmetic and logic operations such as $+$, $\times$, $\land$, $\lor$, and $\neg$. Compared to CPUs, GPUs comprise much simpler compute units and thousands of cores, which are highly focused on (one could say monotonous) parallel data processing rater than convoluted tasks performed sequentially. While each thread, with support for fp and int, generally handles one operation (e.g. $+$ or $\times$) per cycle, cuda cores are not yet optimal for matrix multiplication. Consequently, other neighboring hardware exists to remedy their limitations and increase overall throughput.

NVIDIA introduced [tensor cores]() in its Volta and Turing architectures (2017). It primarily aims to compute $4 \times 4$ matrix multiplication, called [fused multiply-add]() (FMA) operations, per clock cycle in [mixed precision](). Input matrices for tensor cores are represented in fp16 (i.e. 2 bytes per element) that is half the precision of its output, and larger NNs can require i) less memory for training and deployment, ii) less memory bandwidth, and iii) faster mathematical operations. This precision reducing should be selectively applied only in parts of the computation that do not require full precision to maintain model quality.

- Pytorch implemented the Automatic Mixed Precision (AMP)
  - high precision operation: sigmoid, tanh, ... => fp32
  - low precision operation: convolution, pooling, ... => fp16

*The more memory hierarchy is optimised, the higher data throughput and lower latency*. SM in A100 has its own 192 KB of [shared memory]() (SMEM) and SMEM provides high bandwidth and low latency (i.e. close to the compute units), that are desirable for large matrix multiplications and convolutions. Whereas, [L1 cache]() reduces the need for accessing slower global memory as it holds frequently used data. NVIDIA significantly improved performance of their Volta (V100, the predecessor of the A100) by combining these two memories and allowing L1 cache operations to attain the benefits of SMEM. A CUDA programmer is still free from complex memory management via APIs.

Next, there is [L2 cache]() feeding massive computational throughput to L1/SMEM. It is shared across all SMs and provide fast access to larger datasets and helping manage the flow between SMs and the GPU’s [high-bandwidth memory]() (HBM2). The term HBM2 is often interchangably used with GPU memroy, GPU RAM, DRAM, and VRAM. It is the framebuffer size of the GPU and DRAM sitting in the GPU.

- Data (i.e. matrix) being copied from main memory into cache will not be contiguous in cache, but cuda does contiguous allocations for the same data (stored in main mem) in shared mem, thus the latter is more of what's called a scratch pad.
- A scratch pad is a different address space, while cache is just an implementation detail on the same address space (i.e. of RAM)
- Summary of key technologies enabling HBM: TSVs, silicon interposer, thermal management solutions, and etc.

An [interconnect]() is a communication link that enables data transfer between components within a computer system. In the context of GPUs and CPUs, interconnects allow different parts of a system—like processors, memory, storage devices, and expansion cards—to communicate efficiently. They can be physical connections like cables or circuit pathways on a motherboard, or even software protocols that manage how data flows between components. For example, technologies like NVIDIA’s NVLink is an [inter-GPU and GPU-CPU Interconnects](), managing data transfer across various types of multiple processing units, and allowing GPUs to efficiently share data for large-scale parallel processing tasks. Whereas, [system interconnects]()link the broader components of a system, such as processors, memory, storage, and expansion cards. Common interfaces like [peripheral component interconnect express]() (PCIe) supports high-throughput connections essential for tasks that depend on rapid access to storage or network resources, maintaining data flow throughout the entire system.
